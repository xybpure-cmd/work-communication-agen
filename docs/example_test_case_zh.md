# 示例测试用例（中文，V1.0）

下面是一个**可直接复制到页面中**的 V1.0 测试用例。  
目标：验证 4 个输入框可正确生成 1 个输出草案。

## 输入

### 1）收到的消息（可空）
客户问：这个功能这周内可以上线吗？我们需要今天给内部答复。

### 2）沟通背景
- 目前开发完成约 80%，核心功能可用。
- 还剩联调与回归测试，至少需要 2 个工作日。
- 本周还有一个高优先级缺陷在处理中。

### 3）参考材料（可选）
- 项目排期：联调 1 天，回归测试 1 天。
- 已知风险：第三方接口晚高峰偶发超时。
- 团队结论：在无新增阻塞下，最早可在下周一中午前提供上线结论。

### 4）我的意图
不盲目承诺本周上线；先同步当前进度与风险，再给出明确的下一次确认时间点，让对方可向内部同步预期。

## 预期输出（检查标准）

点击生成后，应出现 **1 个输出框**：

- 一版回复草案

草案内容应满足：

1. 语气专业、礼貌。
2. 不直接承诺“本周一定上线”。
3. 能说明当前进展 + 风险点。
4. 给出明确后续时间点（例如“下周一中午前确认”）。
5. 内容是可直接发送的回复草案，不包含“建议如下”“可按以下方向推进”等分析性表达。

> 说明：V1.0 不提供多版本输出，也不需要选择联系人/事项。

## 可选字段校验（参考材料留空）

将「参考材料」留空后再次点击生成，页面仍应正常产出「一版回复草案」。
